SAM1-Prompt Route vs SAM3 Grounding Route (Publication Schematic)

Both routes start from the same image backbone cache. The SAM1 interactive branch then injects geometric prompts into SAM prompt encoder/mask decoder, while normal SAM3 grounding fuses text and geometry into detector-style encoder/decoder prompts.

Why SAM1 prompt mode aligns well with Stage-1 distilled image features

  • Stage-1 distillation supervises the pre-FPN visual substrate used downstream by the interactive path, so both teacher/student preserve the same core spatial semantics.
  • Interactive prompting is additive: prompt tensors condition decoding, but image embeddings remain the dominant carrier of object boundaries and region evidence.
  • COCO evaluation uses strong geometric priors (box prompts), which reduces ambiguity and makes moderate embedding mismatch less harmful.
  • In code, both paths reuse cached backbone outputs from set_image before branching into either interactive or grounding-specific prompt fusion.

A) COCO Eval Path with SAM1 Interactive Prompting

Execution chain: eval_coco -> Sam3Processor.set_image -> _prepare_interactive_features -> predict_inst -> SAM3InteractiveImagePredictor.

1Input image and preprocessing
Image is resized/normalized in set_image and passed to forward_image.
image: [1, 3, 1008, 1008] state.orig_hw: (H0, W0)
↓
2Backbone cache (shared visual substrate)
backbone.forward_image builds multi-scale features and stores backbone_out.
P0: [1, C0, 288, 288] P1: [1, C1, 144, 144] P2: [1, C2, 72, 72]
↓
3Interactive feature adapter
_prepare_interactive_features applies conv_s0/conv_s1 and writes sam2_backbone_out.
sam2_backbone_out.backbone_fpn spatial: 288/144/72 preserved
↓
4predict_inst feature packing
Backbone tensors are repacked for interactive predictor as image_embed + high_res_feats.
image_embed: [1, Ce, 72, 72] high_res_feats: [1, Ch, 288, 288], [1, Ch, 144, 144]
↓
5Prompt tensor construction (_prep_prompts)
Box/point prompts are normalized to model coordinates and concatenated into prompt tokens.
box_coords: [B, 2, 2] point_coords: [B, N, 2] point_labels: [B, N]
↓
6Prompt encoder + SAM mask decoder
sam_prompt_encoder outputs sparse/dense prompts; sam_mask_decoder fuses prompts with image_embed/high_res_feats.
sparse_prompt: [B, Np, D] low_res_masks: [B, Cm, 256, 256] iou_pred: [B, Cm]
↓
7Postprocess to original image size
low_res masks are upsampled to (H0, W0), thresholded, and scored for COCO IoU evaluation.
masks: [B, Cm, H0, W0] scores: [B, Cm]

B) Normal SAM3 Grounding (Text + Geometry)

Execution chain: set_image -> set_text_prompt/add_geometric_prompt -> _forward_grounding -> _encode_prompt -> _run_encoder/_run_decoder.

1Same image preprocessing and backbone cache
set_image uses the same forward_image backbone pipeline as the interactive route.
image: [1, 3, 1008, 1008] backbone_fpn levels cached
↓
2Prompt entry (text and/or geometry)
forward_text adds language_features/language_mask; add_geometric_prompt appends normalized boxes.
language_features: [T, B, D] language_mask: [B, T] geo_box: [B, G, 4]
↓
3_encode_prompt: geometry encoder + concat
geo_feats/geo_masks are built from geometric prompts and concatenated with text prompts.
geo_feats: [G, B, D] prompt: [T + G, B, D] prompt_mask: [B, T + G]
↓
4_run_encoder: image memory conditioned by prompt
Transformer encoder ingests image pyramid with prompt tokens and returns encoded memory.
encoder_hidden_states: [Nimg, B, D] spatial_shapes: [[288,288],[144,144],[72,72],...]
↓
5_run_decoder: object queries -> class/box logits
Decoder cross-attends to encoded memory and prompt context to output object-level predictions.
queries: [B, Q, D] pred_logits: [B, Q, Ccls] pred_boxes: [B, Q, 4]
↓
6_run_segmentation_heads
Segmentation heads consume backbone_fpn + decoder queries to produce per-query masks.
pred_masks: [B, Q, Hm, Wm] pred_boxes_xyxy: [B, Q, 4]
↓
7Processor postprocess
Thresholding, box conversion, and resize to original coordinates for final masks/boxes/scores.
masks: [K, H0, W0] boxes: [K, 4] scores: [K]
Interpretation: The strongest shared interface is the image feature cache produced right after set_image. Stage-1 image distillation improves this interface directly; SAM1 interactive prompting then adds geometric constraints at decode time, which is why interactive box/point tasks can remain robust even when student-teacher trunk metrics are not perfect.
